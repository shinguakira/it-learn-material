# タスク6：実際のレストランWebサイト（オプション）

[← 初級タスクに戻る](../index.md)

> 参照：[HTML](../../../frontend/html/) | [CSS](../../../frontend/css/) | [TypeScript（JavaScript）](../../../frontend/typescript/)

学んだことを活かして、プロフェッショナルなレストランWebサイトをカスタマイズしましょう。完全なコードが用意されています。あなたの仕事はそれを自分のものに変更することです。

「KUROYA」という高級和食レストランのWebサイトです。今まで学んだ技術に加えて新しい技術も使われています。すべてを理解する必要はありません。目標は既存のコードを読んで変更することです。

## セットアップ

新しいフォルダ（例：`restaurant/`）に以下の3つのファイルを作成してください。

コード（HTML・CSS・JavaScript）は英語版と同じです。英語版からコピーしてください：
[英語版のコード](../../../../en/learning-guide/beginner/1-basic-frontend/6-real-restaurant-website.md)

3つのファイルを作成します：
- `index.html` — HTMLコード
- `style.css` — CSSコード
- `script.js` — JavaScriptコード

## タスク

レストランWebサイトが動作することを確認したら、カスタマイズしましょう：

### レストランを変更する（必須）

- [ ] レストラン名を「KUROYA」から自分のレストラン名に変更する（HTMLの中で出てくるすべての場所を見つけること — ロゴ、ヒーロー、ローダー、フッター、タイトル）
- [ ] ヒーローのサブタイトル（例：「Contemporary Japanese Cuisine」）を自分のレストランに合わせて変更する
- [ ] ヒーローセクションのロケーションテキストを変更する
- [ ] メニューの料理を変更する — 各カテゴリ（おまかせ、前菜、メイン、デザート）の名前、説明、値段を更新する
- [ ] Aboutセクションを自分のストーリーとシェフ名に更新する
- [ ] Infoセクションを自分の営業時間、住所、連絡先に更新する
- [ ] フッターの著作権テキストを自分のレストラン名に更新する

### スタイルを変更する（必須）

- [ ] ゴールドカラーを変更する — `style.css`の`:root`セクションで`--gold: #c9a96e`を自分の好きな色に置き換える
- [ ] ダークな背景色を変更する — `--bg-dark`を別の色に変えてみる（例：ブルーダークの`#0b0e14`）
- [ ] フォントを変更する — `'Cormorant Garamond'`を別のGoogle Fontに置き換えてみる

### 機能を追加する（必須）

- [ ] 任意のカテゴリにメニューを最低2つ追加する
- [ ] 新しいギャラリーアイテムを追加する（既存の`.gallery-item`のdivをコピーして色やテキストを変更する）
- [ ] 新しいレビューを追加する（既存の`.testimonial`のdivをコピーして、対応する`.testimonial-dot`ボタンも追加する）

### ボーナス（オプション）

- [ ] メニュータブの名前を変更する（例：「Omakase」→「ランチセット」、「Appetizers」→「前菜」）
- [ ] 新しいメニューカテゴリタブを追加する（新しい`<button class="menu-tab">`と、新しい`data-category`を持つメニューアイテムを追加する）
- [ ] ギャラリーのグリッドを4列から3列に変更する
- [ ] CTAボタンのリンク先を別のURLに変更する
- [ ] ヒーローのアニメーション速度を変えてみる（CSSの`animation`プロパティを探す）

## 理解すべきポイント

このWebサイトを読んで修正した後、以下を理解できるはずです：

- **CSS変数**（`--gold`、`--bg-dark`など）で色を一か所で変更するだけで全体に反映される
- **`position: fixed`**でスクロール時もナビバーを表示し続ける
- **`display: grid`**と`grid-template-columns`でレイアウトを作る（メニュー、情報、ギャラリー）
- **`data-*`属性**（`data-tab`、`data-category`）でHTMLにカスタムデータを保存し、JavaScriptで読み取る
- **CSSアニメーション**（`@keyframes`）でローディングやフェードイン効果を作る
- **`IntersectionObserver`**で要素がスクロールして画面に入ったことを検出する（リビールアニメーションに使用）
- **`classList.add()` / `classList.remove()`**でJavaScriptからCSSクラスを切り替える
- **`setInterval()`**でコードを繰り返し実行する（レビューの自動ローテーションに使用）
- **`e.preventDefault()`**でフォームがページをリロードするのを防ぐ
